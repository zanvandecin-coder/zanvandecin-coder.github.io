### Инструкция по работе с Markdown

Markdown — это простой язык разметки для форматирования текста. Он читается и редактируется как обычный текст, но может быть преобразован в HTML и другие форматы. Разберём базовый синтаксис пошагово.

#### 1. Заголовки

Используйте символ решётки (`#`). Количество решёток определяет уровень заголовка (от H1 до H6):

markdown

```markdown
# Заголовок первого уровня (H1)
## Заголовок второго уровня (H2)
### Заголовок третьего уровня (H3)
#### Заголовок четвёртого уровня (H4)
##### Заголовок пятого уровня (H5)
###### Заголовок шестого уровня (H6)
```

**Важно:** оставляйте пробел между решётками и текстом заголовка.

#### 2. Выделение текста

- **Жирный текст:** используйте две звёздочки `**` или два нижних подчёркивания `__`:

  markdown

```markdown
**Жирный текст** или __Жирный текст__
```

*Курсив:* используйте одну звёздочку `*` или одно нижнее подчёркивание `_`:

markdown

```markdown
*Курсив* или _Курсив_
```

Зачёркнутый текст (доступно в расширенных версиях Markdown, например, GitHub Flavored Markdown): используйте две тильды `~~`:

markdown

```markdown
~~Зачёркнутый текст~~
```

***Жирный курсив:*** комбинируйте символы:

markdown

- ```markdown
  ***Жирный курсив*** или __**Жирный курсив**_
  ```

#### 3. Списки

- **Маркированный список:** используйте `-`, `+` или `*`:

  markdown

```markdown
- Пункт 1
- Пункт 2
  - Подпункт 2.1
  - Подпункт 2.2
- Пункт 3
```

**Нумерованный список:** используйте цифры с точкой:

markdown

- ```markdown
  1. Первый пункт
  2. Второй пункт
     1. Подпункт 2.1
     2. Подпункт 2.2
  3. Третий пункт
  ```

#### 4. Цитаты

Используйте символ `>` перед каждой строкой цитаты:

markdown

```markdown
> Это цитата.
> > Это вложенная цитата.
```

#### 5. Горизонтальные линии

Вставьте три или более одинаковых символа на отдельной строке: `***`, `---` или `___`:

markdown

```markdown
***
---
___
```

#### 6. Ссылки

- **Встроенные ссылки:**

  markdown

```markdown
[Текст ссылки](https://www.example.com)
```

**Ссылки с подсказкой:**

markdown

```markdown
[Текст ссылки](https://www.example.com "Подсказка при наведении")
```

**Ссылки-сноски** (удобны для длинных URL):

markdown

- ```markdown
  Текст с [ссылкой][id]
  
  [id]: https://www.example.com "Заголовок ссылки"
  ```

#### 7. Изображения

Синтаксис похож на ссылки, но с восклицательным знаком в начале:

markdown

```markdown
![Альтернативный текст](путь/к/изображению.jpg "Подсказка")
```

#### 8. Код

- **Инлайн-код** (в строке): оберните текст в обратные кавычки ```:

  markdown

```markdown
Используйте функцию `print()` для вывода.
```

**Блок кода:** оберните код в три обратные кавычки ` `. Можно указать язык программирования для подсветки синтаксиса:

markdown

~~~markdown
```python
def hello():
    print("Hello, World!")
~~~

- 

#### 9. Таблицы (в расширенных версиях Markdown)

Используйте вертикальные черты `|` и дефисы `-` для разделения столбцов и строк:

markdown

```markdown
| Заголовок 1 | Заголовок 2 | Заголовок 3 |
|------------|------------|------------|
| Ячейка 1   | Ячейка 2   | Ячейка 3   |
| Ячейка 4   | Ячейка 5   | Ячейка 6   |
```

#### 10. Списки задач (в GitHub Flavored Markdown)

Создавайте чек‑боксы для отслеживания задач:

markdown

```markdown
- [ ] Невыполненная задача
- [x] Выполненная задача
```

#### 11. Экранирование символов

Если нужно вывести служебный символ Markdown как обычный текст, поставьте перед ним обратный слэш `\`:

markdown

```markdown
\*Это не курсив, а звёздочка\*
```

### Инструменты для работы

1. **Онлайн‑редакторы:**
   - Dillinger;
   - Markdown Here;
   - StackEdit.
2. **Десктопные редакторы:**
   - Typora (простой и мощный);
   - iA Writer (для macOS);
   - Atom (с поддержкой расширений).
3. **Интеграция в платформы:**
   - GitHub/GitLab (документация, README);
   - Notion/Obsidian (заметки);
   - Jekyll/Hugo (блоги).

------

### Советы

- Проверяйте рендеринг: разные платформы могут поддерживать разные версии Markdown.
- Используйте пустые строки для разделения блоков текста — это улучшает читаемость исходного кода.
- Сохраняйте файлы с расширением `.md` или `.markdown`.
